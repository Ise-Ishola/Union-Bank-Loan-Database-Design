# Union Bank Database Design (SQL)

This project focuses on designing a relational database for a union bank loan management system using SQL. The database was created to support structured storage and retrieval of loan-related data while maintaining data integrity through normalized tables, primary keys, foreign keys, and constraints.

## Project Overview
The goal of this project was to design a database schema for a banking loan system that can efficiently manage borrowers, loans, and related financial records.

## Key Features
- Designed normalized relational tables
- Defined primary and foreign key relationships
- Applied constraints to preserve data integrity
- Structured the database for efficient data storage and retrieval

## Tools Used
- SQL
- Relational Database Design
- Data Modeling

## File
- Union-Bank-Loan-Database-Design.sql

## Outcome
This project demonstrates foundational SQL and database design skills, including schema creation, table relationships, and data integrity enforcement for a banking use case.

## Stakeholder Feedback
Stakeholders noted that the database design created a reliable structure for storing and managing borrower and loan information. The normalized schema and clearly defined table relationships improved data consistency and made it easier to maintain accurate financial records. The structure also supports scalable querying for reporting and analytical use cases.
